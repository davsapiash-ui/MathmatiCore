name: spec_updater
description: A mandatory skill for updating specification (Spec) documents whenever there is an architectural, logic, or data flow change.
---

# מטרת הסקיל
סקיל זה נועד למנוע "ריקבון תוכנה" (Software Rot) ולהבטיח שכל הסוכנים עובדים על בסיס מסמכי אפיון (Specs) מעודכנים. הסקיל מחליף מסמכים ישנים בחדשים תוך תיעוד מדויק של זמן העדכון.

# הנחיות להפעלת הסקיל:
כאשר אתה מבצע שינוי ארכיטקטוני או משנה לוגיקה במערכת, עליך לבצע את השלבים הבאים באמצעות סקיל זה:

1. **איתור המסמך הישן:** אתר את מסמך האפיון הרלוונטי בתיקיית `מסמכי אפיון/`.
2. **יצירת מסמך חדש והטמעת חותמת זמן:** 
   - צור את המסמך המעודכן עם תאריך ושעה של העדכון האחרון בראש המסמך, בפורמט הבא: `> **תאריך עדכון אחרון: DD.MM.YYYY HH:MM**`
3. **החלפה ומחיקה:** 
   - אל תשאיר כפילויות! ודא שהמסמך הישן נמחק או נדרס במלואו על ידי המסמך החדש.
   - השתמש ב-`write_to_file` עם `Overwrite: true` או מחק את הישן מפורשות.
4. **דחיפה (Deploy):** בצע Commit המציין "docs: updated spec document via spec_updater skill" ודחוף ל-GitHub כדי שכלל הסוכנים יסתנכרו עליו.
